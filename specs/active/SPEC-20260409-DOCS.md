# Agent #1 — Orchestrator

## 1. Agent Overview

## 2. Agents & Capabilities
[Table with all 4 agents]
| Role | Type | LLM | Inputs | Outputs | Tools |
|------|------|-----|--------|---------|-------|
| Orchestrator | Semi-autonomous (HITL on ticket create) | MiniMax M2.7 + 3-model fallback | Text, images, PDFs, log files | Streaming triage cards, ticket preview/confirmed states | triageWorkflow (tool), 8 registered tools total |

## 3. Architecture & Orchestration
[Keep Mermaid diagram, update Security Pipeline node to note it is planned/partial]
- Orchestration approach: Sequential Mastra durable workflow. 8 steps: intake → triage
  → dedup → ticket (two-phase: preview then confirm) → notify → suspend → verify →
  notify-resolution.
- State management: Mastra workflow state in LibSQL. Chat state via AI SDK useChat +
  MemoryLibSQL per thread. Workflow survives container restarts.
- Error handling: Tool-level error boundaries (try/catch in each tool). Graceful
  degradation: Linear unavailable → local_tickets table; Resend down → log and skip;
  OpenRouter → fallback model chain.
- Handoff: Orchestrator → triageWorkflow (Mastra tool call) → Triage Agent (step 2)
  → [suspend] → Resolution Reviewer (webhook resume, step 6). All handoffs use typed
  Zod-validated data.

## 4. Context Engineering
- Sources: user text + attachments (images/PDFs/logs), codebase wiki chunks (RAG),
  Linear ticket history (dedup), per-thread conversation memory (MemoryLibSQL)
- Strategy: Two-pass llm-wiki. Pass 1: per-file summaries (purpose, key functions,
  dependencies). Pass 2: cross-module synthesis. Stored as F32_BLOB(1536) vectors.
  Retrieval: top-10 chunks via DiskANN index (vector_top_k).
- Note: wiki ingestion pipeline exists (wiki-generate.ts, wiki-query.ts) but RAG data
  not yet populated; queryWikiTool returns [] gracefully.
- Token management: chunk-level retrieval (not full documents), top-10 cap. Images
  resized client-side via Canvas API. File limits: 10MB/file, 25MB/message.
- Grounding: structured output via triageOutputSchema prevents hallucinated fields.
  Confidence scoring (0-1) with low-confidence warning UI component at threshold.
