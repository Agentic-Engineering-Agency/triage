# SPEC-20260408-002: Better Auth + Drizzle/LibSQL Authentication

**ID:** SPEC-20260408-002
**Name:** better-auth-db-auth
**Stage:** SPEC
**Created:** 2026-04-08
**Updated:** 2026-04-08
**Author:** Chenko

## Purpose

Implement session-based authentication for the Triage SRE agent using Better Auth with a Drizzle ORM adapter against LibSQL self-hosted (sqld v0.24.32 in Docker). The auth layer covers: Drizzle client singleton, Better Auth instance with Drizzle adapter (provider `'sqlite'`), schema tables for user/session/account/verification, drizzle-kit push workflow, Zod validators via `drizzle-orm/zod`, `/auth/*` route mounting in Mastra/Hono, `createAuthClient` in the React frontend, and route guards. Session expires at fixed 7 days (no sliding window) with HttpOnly + SameSite=Lax cookies.

## Scope

### In Scope

- Drizzle ORM client singleton (`runtime/src/db/client.ts`) using `drizzle-orm/libsql` + `@libsql/client` pointing to `http://libsql:8080`
- `drizzle.config.ts` with `dialect: 'turso'` and `drizzle-kit push` workflow
- Auth schema tables: `user`, `session`, `account`, `verification` (Better Auth required columns) in `runtime/src/db/schema.ts`
- `drizzle-kit push` to apply schema to LibSQL
- Better Auth instance in `runtime/src/auth/index.ts` with `drizzleAdapter(db, { provider: 'sqlite' })`
- Session config: 7-day fixed expiry, no sliding window, HttpOnly + SameSite=Lax cookies
- Same-origin `/auth/*` route mounting on Mastra's underlying Hono server via Caddy reverse proxy
- Frontend `createAuthClient` in `frontend/src/lib/auth-client.ts`
- `useAuth` hook replacing the existing stub in `frontend/src/hooks/use-auth.ts`
- Zod schemas for auth tables via `createSelectSchema`/`createInsertSchema` from `drizzle-orm/zod`
- Route guards for protected API endpoints

### Out of Scope

- OAuth/social providers (GitHub, Google, etc.) — email+password only for hackathon
- Two-factor authentication (2FA)
- Password reset email flows
- Organization/multi-tenancy features
- JWT/bearer token auth (session cookies only)
- Rate limiting configuration
- Migration files (use `drizzle-kit push` only)
- Non-Drizzle database adapters (Prisma, Mongoose, etc.)

### Ticket Alignment (INFRA-02 — Better Auth Setup)

- Register flow coverage: REQ-A04, REQ-A06, REQ-A08, REQ-A09
- Login flow coverage: REQ-A04, REQ-A06, REQ-A08, REQ-A09
- Session persistence across page refresh: REQ-A05, REQ-A08, REQ-A09, REQ-A10

## Requirements

### REQ-A01: Drizzle Client Singleton

**Priority:** P0
**Description:** The runtime SHALL provide a singleton Drizzle client connected to LibSQL at `http://libsql:8080` using `drizzle-orm/libsql`.

#### Acceptance Criteria

- **GIVEN** the runtime container starts **WHEN** `runtime/src/db/client.ts` is imported **THEN** a Drizzle client connected to LibSQL via `@libsql/client` HTTP protocol SHALL be exported
- **GIVEN** `LIBSQL_URL` env var is set to `http://libsql:8080` **WHEN** the client initializes **THEN** it SHALL connect to that URL without an authToken

#### Scenarios

- **Happy path:** Client initializes successfully, queries execute against LibSQL
- **Edge case:** `LIBSQL_URL` env var is missing → falls back to default `http://libsql:8080`
- **Error case:** LibSQL server unreachable → connection throws, application logs error and fails health check

---

### REQ-A02: drizzle.config.ts with turso Dialect

**Priority:** P0
**Description:** The project SHALL contain a `drizzle.config.ts` using `dialect: 'turso'` pointing to the LibSQL URL.

#### Acceptance Criteria

- **GIVEN** `drizzle.config.ts` exists at the project root **WHEN** `npx drizzle-kit push` runs **THEN** it SHALL use `dialect: 'turso'`
- **GIVEN** the schema file is at `runtime/src/db/schema.ts` **WHEN** drizzle-kit parses the config **THEN** it SHALL find all exported tables

#### Scenarios

- **Happy path:** `drizzle-kit push` connects to LibSQL and applies schema without errors
- **Edge case:** `LIBSQL_URL` env var points to wrong URL → push fails with connection error
- **Error case:** Schema has syntax errors → drizzle-kit reports parse error and exits non-zero

---

### REQ-A03: Auth Schema Tables

**Priority:** P0
**Description:** The runtime SHALL define Drizzle schema tables for Better Auth's required data model in `runtime/src/db/schema.ts`: `user`, `session`, `account`, `verification`.

#### Acceptance Criteria

- **GIVEN** `runtime/src/db/schema.ts` exists **WHEN** `drizzle-kit push` runs **THEN** four tables SHALL be created in LibSQL: `user`, `session`, `account`, `verification`
- **GIVEN** the `user` table **WHEN** it is created **THEN** it SHALL contain: `id` (text PK), `name` (text), `email` (text unique), `emailVerified` (integer/boolean), `image` (text nullable), `createdAt` (integer/timestamp), `updatedAt` (integer/timestamp)
- **GIVEN** the `session` table **WHEN** it is created **THEN** it SHALL contain: `id` (text PK), `userId` (text FK→user), `expiresAt` (integer/timestamp), `token` (text unique), `ipAddress` (text nullable), `userAgent` (text nullable), `createdAt`, `updatedAt`
- **GIVEN** the `account` table **WHEN** it is created **THEN** it SHALL contain provider fields (`accountId`, `providerId`, `accessToken`, `refreshToken`, `idToken`, scope, password, etc.)
- **GIVEN** the `verification` table **WHEN** it is created **THEN** it SHALL contain: `id`, `identifier`, `value`, `expiresAt`, `createdAt`, `updatedAt`

#### Scenarios

- **Happy path:** All 4 tables created with correct columns and constraints via `drizzle-kit push`
- **Edge case:** Table already exists with matching schema → drizzle-kit push succeeds (no-op or alter)
- **Error case:** Table exists with incompatible schema (e.g., different column type) → drizzle-kit push fails with schema mismatch error

### REQ-A04: Better Auth Instance with Drizzle Adapter

**Priority:** P0
**Description:** The runtime SHALL export a Better Auth instance from `runtime/src/auth/index.ts` configured with `drizzleAdapter(db, { provider: 'sqlite' })`.

#### Acceptance Criteria

- **GIVEN** `runtime/src/auth/index.ts` is imported **WHEN** it initializes **THEN** it SHALL call `drizzleAdapter(db, { provider: 'sqlite' })` as the database adapter
- **GIVEN** `emailAndPassword.enabled: true` **WHEN** a user submits email+password **THEN** Better Auth SHALL create a user record and session
- **GIVEN** `BETTER_AUTH_SECRET` is at least 32 characters **WHEN** Better Auth starts **THEN** it SHALL use it as the signing secret

#### Scenarios

- **Happy path:** Better Auth starts, generates signed session tokens, creates user/session records in LibSQL
- **Edge case:** `BETTER_AUTH_SECRET` is shorter than 32 chars → Better Auth logs a warning but proceeds
- **Error case:** DB connection fails → Better Auth throws on first request, not on import

### REQ-A05: Session Cookie Configuration

**Priority:** P0
**Description:** Session cookies SHALL be configured for 7-day fixed expiry with HttpOnly, SameSite=Lax, and Secure in production.

#### Acceptance Criteria

- **GIVEN** a user logs in **WHEN** a session is created **THEN** the session cookie SHALL have `expiresIn: 60 * 60 * 24 * 7` (7 days)
- **GIVEN** session cookies **WHEN** they are created **THEN** they SHALL have `httpOnly: true`, `sameSite: 'lax'`, and `secure: true` (only when `NODE_ENV === 'production'`)
- **GIVEN** sliding window is disabled **WHEN** a user makes requests **THEN** the session expiry SHALL NOT be extended (fixed expiry at creation time)

#### Scenarios

- **Happy path:** User logs in, cookie set with 7-day expiry, user can access protected routes for 7 days without re-auth
- **Edge case:** `NODE_ENV=development` → `secure: false` on cookie, session still works on HTTP localhost
- **Error case:** Cookie rejected by browser → check SameSite and Secure attribute compatibility with browser

### REQ-A06: /auth/* Route Mounting in Mastra/Hono

**Priority:** P0
**Description:** The runtime SHALL mount Better Auth's HTTP handler at `/auth/*` on Mastra's underlying Hono server, preserving the project's same-origin Caddy proxy architecture.

#### Acceptance Criteria

- **GIVEN** Mastra starts on port 4111 **WHEN** a POST or GET request arrives at `/auth/*` **THEN** it SHALL be routed to `auth.handler(c.req.raw)`
- **GIVEN** Caddy reverse proxy is configured **WHEN** a browser request goes to `http://localhost:3001/auth/*` **THEN** it SHALL be forwarded unchanged to `runtime:4111/auth/*`
- **GIVEN** the browser only talks to the Caddy origin **WHEN** auth requests are made through `/auth/*` **THEN** no dedicated CORS configuration SHALL be required for Better Auth session cookies to work

#### Scenarios

- **Happy path:** `POST /auth/sign-in` creates user+session and returns session cookie; subsequent GET `/auth/get-session` returns user data
- **Edge case:** Browser page refresh after login → same-origin cookie remains attached and `get-session` returns the authenticated user
- **Error case:** Request to `/auth/*` with missing body → Better Auth returns 400 with validation error

---

### REQ-A07: drizzle-kit Push Workflow

**Priority:** P0
**Description:** Schema changes SHALL be applied using `drizzle-kit push` (no migration files).

#### Acceptance Criteria

- **GIVEN** schema files exist in `runtime/src/` and `drizzle.config.ts` is at the repository root **WHEN** `npx drizzle-kit push` executes from the repository root **THEN** all tables SHALL be created or updated in LibSQL without generating migration files
- **GIVEN** `drizzle-kit push` succeeds **WHEN** it completes **THEN** it SHALL exit with code 0 and log the changes applied

#### Scenarios

- **Happy path:** New table added → drizzle-kit push creates it; existing table modified → drizzle-kit push alters it
- **Edge case:** No schema changes since last push → drizzle-kit push reports "nothing to push"
- **Error case:** Schema references missing dependency → drizzle-kit push fails with foreign key error

### REQ-A08: Frontend createAuthClient

**Priority:** P0
**Description:** The frontend SHALL use Better Auth's `createAuthClient` for session management.

#### Acceptance Criteria

- **GIVEN** `frontend/src/lib/auth-client.ts` **WHEN** it is imported **THEN** it SHALL configure `createAuthClient` for same-origin browser requests using the current browser origin and SHALL NOT hardcode `http://localhost:3001`
- **GIVEN** the auth client **WHEN** it makes requests **THEN** it SHALL use `fetchOptions: { credentials: 'include' }` to send auth cookies
- **GIVEN** the frontend makes a request to `/auth/*` **WHEN** the browser sends the request **THEN** the auth cookie SHALL be included automatically

#### Scenarios

- **Happy path:** `authClient.getSession()` returns user data when session cookie is valid
- **Edge case:** No session cookie present → `getSession()` returns null (no error)
- **Error case:** Network error → `getSession()` rejects with network error

---

### REQ-A09: useAuth Hook

**Priority:** P0
**Description:** The frontend SHALL provide a `useAuth` hook that returns user, isLoading, and isAuthenticated state.

#### Acceptance Criteria

- **GIVEN** `useAuth()` is called in a React component **WHEN** the component renders **THEN** it SHALL return `{ user, isLoading, isAuthenticated }` where `isAuthenticated` is `true` iff `user !== null`
- **GIVEN** the hook **WHEN** a session exists **THEN** `user.id`, `user.email`, `user.name` SHALL be available
- **GIVEN** the hook **WHEN** no session exists **THEN** `user` SHALL be `null` and `isAuthenticated` SHALL be `false`

#### Scenarios

- **Happy path:** User authenticated → hook returns full user object with id, email, name
- **Edge case:** Session loading → `isLoading: true` while session is being fetched
- **Error case:** Session expired → user becomes null, `isAuthenticated: false`

---

### REQ-A10: Protected Route Guard

**Priority:** P0
**Description:** The frontend SHALL provide a guard function to protect routes that require authentication.

#### Acceptance Criteria

- **GIVEN** `requireAuth()` is called **WHEN** a valid session exists **THEN** it SHALL return the session object
- **GIVEN** `requireAuth()` is called **WHEN** no session exists **THEN** it SHALL throw an error with message `'Unauthorized'`

#### Scenarios

- **Happy path:** Protected page load → `requireAuth()` returns session, component renders
- **Edge case:** Session expires mid-session → next navigation attempt throws, redirect to login
- **Error case:** `requireAuth()` called outside React lifecycle → throws without crash

---

### REQ-A11: Zod Schemas for Auth Tables

**Priority:** P1
**Description:** The runtime SHALL provide Zod schemas for auth tables using `createSelectSchema` and `createInsertSchema` from `drizzle-orm/zod`.

#### Acceptance Criteria

- **GIVEN** `runtime/src/lib/schemas/auth.ts` **WHEN** it is imported **THEN** it SHALL export `userSelectSchema`, `userInsertSchema`, `sessionSelectSchema`, `accountSelectSchema`, `verificationSelectSchema`
- **GIVEN** `drizzle-orm/zod` is used **WHEN** schemas are generated **THEN** `drizzle-zod` package SHALL NOT be imported

#### Scenarios

- **Happy path:** Schema imported and used in Zod validation → request/response typed correctly
- **Edge case:** Table schema changes → Zod schema auto-generated from Drizzle reflects changes
- **Error case:** Missing required column in schema → compile-time TypeScript error

---

### REQ-A12: Auth Middleware on Protected Routes

**Priority:** P1
**Description:** The runtime SHALL provide a Hono middleware that injects `user` and `session` variables into the context for all routes.

#### Acceptance Criteria

- **GIVEN** a request to any protected route **WHEN** the middleware runs **THEN** it SHALL call `auth.api.getSession({ headers })` and set `c.set('user', session?.user ?? null)` and `c.set('session', session?.session ?? null)`
- **GIVEN** no session exists **WHEN** a request hits a protected route **THEN** `c.get('user')` SHALL be `null`

#### Scenarios

- **Happy path:** Authenticated request → user and session available in route handler via `c.get('user')`
- **Edge case:** Missing session cookie → session is null, user is null
- **Error case:** Expired session cookie → getSession returns null, user is null

## Technical Approach

### Architecture

```
Frontend (React :3001)
  └── Caddy reverse proxy
        └── /api/*, /auth/* → runtime (:4111)
              └── Mastra (Hono)
                    └── /auth/* → Better Auth handler
                    └── auth.api.getSession() middleware
                          └── LibSQL (sqld :8080)
                                └── user, session, account, verification tables
```

### Stack

| Component | Technology |
|-----------|-----------|
| Database | LibSQL (sqld v0.24.32) in Docker |
| ORM | Drizzle ORM with `drizzle-orm/libsql` |
| DB Config | `dialect: 'turso'` in drizzle.config.ts |
| Schema push | `drizzle-kit push` (no migration files) |
| Auth framework | Better Auth |
| Auth adapter | `@better-auth/drizzle-adapter` with `provider: 'sqlite'` |
| Session | 7-day fixed expiry, HttpOnly, SameSite=Lax |
| Zod | `drizzle-orm/zod` (NOT `drizzle-zod`) |
| HTTP server | Mastra (built on Hono) |
| Frontend auth | `better-auth/react` + `createAuthClient` |

### File Map

```
runtime/src/
  auth/
    index.ts               # Better Auth instance
  db/
    client.ts              # Drizzle singleton
    schema.ts              # Better Auth tables
  lib/
    schemas/
      auth.ts               # Zod schemas from Drizzle

frontend/src/
  lib/
    auth-client.ts          # createAuthClient
  hooks/
    use-auth.ts             # useAuth hook (replaces stub)
  routes/
    (protected routes)       # use requireAuth guard

drizzle.config.ts           # dialect: 'turso'
.env                        # LIBSQL_URL, BETTER_AUTH_SECRET, BETTER_AUTH_URL
```

### Key Constraints

1. `dialect: 'turso'` in drizzle.config.ts (even for sqld local) — NOT `'sqlite'`
2. `provider: 'sqlite'` in drizzleAdapter — NOT `'turso'`
3. `drizzle-orm/zod` — NOT the deprecated `drizzle-zod` package
4. No authToken in LibSQL connection (self-hosted, no auth)
5. `BETTER_AUTH_URL=http://localhost:3001` (browser origin served by Caddy for cookie scope)
6. Same-origin auth only: `/auth/*` is proxied by Caddy; do NOT introduce dedicated auth CORS middleware unless the architecture changes
