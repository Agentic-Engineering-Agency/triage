All environment variables are defined in `.env.example` at the project root. Copy it to `.env` and fill in real values before starting the stack.

```bash
cp .env.example .env
```

Variables marked **Required** cause the runtime to exit on startup if missing. Variables marked **Optional** are validated lazily when the relevant integration is actually used.

## App Configuration

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `PORT` | Optional | `4111` | HTTP port for the Mastra runtime server |
| `NODE_ENV` | Optional | `production` | Runtime environment. Values: `development`, `production`, `test` |
| `LIBSQL_URL` | Optional | `http://libsql:8080` | LibSQL HTTP endpoint. Change if running LibSQL externally |
| `FRONTEND_MODE` | Optional | `static` | Caddy mode. `static` serves built SPA; `dev` proxies to Vite HMR |
| `CADDY_PORT` | Optional | `3001` | Port Caddy listens on |
| `RUNTIME_HOST` | Optional | `runtime` | Hostname Caddy reverse-proxies API requests to |
| `RUNTIME_PORT` | Optional | `4111` | Port Caddy reverse-proxies API requests to |

## Generating Secrets

Several variables require cryptographically strong random values. Use OpenSSL:

```bash
# Single secret
openssl rand -hex 32

# Generate all required secrets at once
for var in BETTER_AUTH_SECRET ENCRYPTION_KEY SALT NEXTAUTH_SECRET \
           LANGFUSE_INIT_PROJECT_PUBLIC_KEY LANGFUSE_INIT_PROJECT_SECRET_KEY \
           CLICKHOUSE_PASSWORD REDIS_AUTH MINIO_ROOT_PASSWORD POSTGRES_PASSWORD \
           LANGFUSE_S3_EVENT_UPLOAD_SECRET_ACCESS_KEY \
           LANGFUSE_S3_MEDIA_UPLOAD_SECRET_ACCESS_KEY \
           LANGFUSE_S3_BATCH_EXPORT_SECRET_ACCESS_KEY; do
  echo "$var=$(openssl rand -hex 32)"
done
```

## Minimum Required Variables

The absolute minimum to start the stack:

```bash
# Required by runtime
OPENROUTER_API_KEY=sk-or-v1-...
BETTER_AUTH_SECRET=<hex>

# Required by Langfuse (without these, langfuse-web won't start)
ENCRYPTION_KEY=<hex>
SALT=<hex>
NEXTAUTH_SECRET=<hex>
CLICKHOUSE_PASSWORD=<password>
REDIS_AUTH=<password>
MINIO_ROOT_PASSWORD=<password>
POSTGRES_PASSWORD=<password>
LANGFUSE_S3_EVENT_UPLOAD_SECRET_ACCESS_KEY=<password>
LANGFUSE_S3_MEDIA_UPLOAD_SECRET_ACCESS_KEY=<password>
LANGFUSE_S3_BATCH_EXPORT_SECRET_ACCESS_KEY=<password>
LANGFUSE_INIT_USER_PASSWORD=<password>
LANGFUSE_INIT_PROJECT_PUBLIC_KEY=<hex>
LANGFUSE_INIT_PROJECT_SECRET_KEY=<hex>
```

Without `LINEAR_API_KEY` or `RESEND_API_KEY`, the application runs in degraded mode — triage works but tickets go to local storage and emails are skipped.
