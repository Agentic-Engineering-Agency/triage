Triage is an AI-powered SRE incident triage agent for Solidus/Rails e-commerce platforms. Engineers describe production incidents in a multimodal chat interface — using text, screenshots, and log snippets — and the agent queries a RAG-indexed codebase wiki to pinpoint root cause, identify affected files, and classify severity.

Once analysis is complete, a triage card is presented for **human approval** before a Linear ticket is created and the on-call team is notified by email. When the fix ships and the Linear ticket is marked Done, a webhook triggers a Resolution Reviewer that independently verifies the fix before closing the loop.

## How It Works

<Steps>
  <Step>
    **Describe the incident** — Send a message in the chat UI with text, screenshots, or log snippets describing what's broken.
  </Step>
  <Step>
    **Automated triage** — The Triage Agent queries the codebase wiki (RAG), cross-references symptoms against source files, and produces a structured severity / confidence / root-cause report.
  </Step>
  <Step>
    **Review and approve** — A triage card shows a preview. You explicitly approve before any Linear ticket is created or email notifications are sent.
  </Step>
  <Step>
    **Resolution verification** — When the fix ships and the Linear ticket is marked Done, a webhook triggers the Resolution Reviewer, which returns a `close / reopen / monitor` recommendation.
  </Step>
</Steps>

## Feature Highlights

- **Multimodal Chat** — Attach screenshots, dashboards, and log files alongside text descriptions. Vision model (Gemma 4 31B) extracts and describes image content before analysis.
- **RAG-Powered Triage** — Codebase wiki indexed as F32_BLOB(1536) vectors in LibSQL. Mercury-2 queries relevant files, cross-references symptoms, and produces structured output.
- **Two-Phase Ticket Creation** — Triage card shows a preview (pending state) requiring explicit user approval before any Linear ticket is created.
- **Linear Integration** — Full ticket lifecycle: create, update, search, and team assignment. Webhook resume triggers resolution review when an issue is marked Done.
- **Email Notifications** — Resend-powered HTML notifications to assignees on ticket creation and to reporters on resolution.
- **Resolution Verification** — Resolution Reviewer agent compares the deployed fix against the original root cause, returning a structured recommendation.
- **Graceful Degradation** — All external services have fallback modes. Triage never blocks on infrastructure failures.
- **Self-Hosted Observability** — Full Langfuse stack (ClickHouse, Redis, MinIO, Postgres) for LLM trace visibility, latency, and cost tracking.
