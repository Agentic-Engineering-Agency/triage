Triage is configured entirely through environment variables defined in `.env`. Copy `.env.example` to `.env` and fill in the values for your environment. Variables marked **Required** will cause the runtime to exit on startup if missing or invalid — validated via Zod at boot time.

## App Configuration

| Variable | Required | Description | Default |
|----------|----------|-------------|---------|
| `PORT` | Optional | HTTP port for the Mastra runtime | `4111` |
| `NODE_ENV` | Optional | Runtime environment (`development`, `production`, `test`) | `production` |
| `LIBSQL_URL` | Optional | LibSQL HTTP endpoint | `http://libsql:8080` |
| `FRONTEND_MODE` | Optional | Caddy mode: `static` (prod, serves built SPA) or `dev` (proxies to Vite HMR) | `static` |
| `CADDY_PORT` | Optional | Port Caddy listens on | `3001` |
| `RUNTIME_HOST` | Optional | Hostname Caddy reverse-proxies to | `runtime` |
| `RUNTIME_PORT` | Optional | Port Caddy reverse-proxies to | `4111` |

## Langfuse Headless Init

These variables auto-create an org, project, and admin user on first boot. After the first startup, they can be removed (the data persists in Postgres).

| Variable | Required | Description | Default |
|----------|----------|-------------|---------|
| `LANGFUSE_INIT_ORG_ID` | Optional | Organization ID to create | `triage-org` |
| `LANGFUSE_INIT_ORG_NAME` | Optional | Organization display name | `Triage SRE` |
| `LANGFUSE_INIT_PROJECT_ID` | Optional | Project ID to create | `triage-project` |
| `LANGFUSE_INIT_PROJECT_NAME` | Optional | Project display name | `Triage Agent` |
| `LANGFUSE_INIT_PROJECT_PUBLIC_KEY` | Optional | Public key to assign to the project. Use this as `LANGFUSE_PUBLIC_KEY` | — |
| `LANGFUSE_INIT_PROJECT_SECRET_KEY` | Optional | Secret key to assign to the project. Use this as `LANGFUSE_SECRET_KEY` | — |
| `LANGFUSE_INIT_USER_EMAIL` | Optional | Admin user email | `admin@agenticengineering.lat` |
| `LANGFUSE_INIT_USER_NAME` | Optional | Admin user display name | `Admin` |
| `LANGFUSE_INIT_USER_PASSWORD` | Optional | Admin user password | — |

## Generating Secrets

Several variables require strong random values. Use OpenSSL to generate them:

```bash
# 32-byte hex string (recommended for most secrets)
openssl rand -hex 32

# Generate multiple at once
for var in BETTER_AUTH_SECRET ENCRYPTION_KEY SALT NEXTAUTH_SECRET; do
  echo "$var=$(openssl rand -hex 32)"
done
```
